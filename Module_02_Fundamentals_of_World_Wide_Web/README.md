# Fundamentals of World Wide Web

## THEORY EXERCISE

### 1. Difference between Web Designer and Web Developer

- A **Web Designer** is responsible for the look and feel of a website, focusing on layout, colors, typography, and overall user experience.
- Designers work with tools like Figma, Adobe XD, or Canva to create visual designs and prototypes.
- A **Web Developer** converts these designs into a working website using programming languages and frameworks.
- Developers handle functionality, performance, databases, and ensure the website works correctly across browsers.

### 2. What is W3C?

- W3C (World Wide Web Consortium) is an international organization that sets standards for the World Wide Web.
- It was founded by Tim Berners-Lee to ensure the long-term growth of the web.
- W3C provides guidelines for HTML, CSS, accessibility, and web security.
- Following W3C standards helps create websites that are compatible, accessible, and reliable.

### 3. What is a Domain?

- A domain is the unique and human-readable address of a website on the internet, such as google.com.
- It is used to locate and identify websites without remembering complex IP addresses.
- Domains are registered through domain registrars and mapped to servers using DNS.
- A domain name helps build brand identity and makes a website easy to access.

### 4. What is SEO?

- SEO (Search Engine Optimization) is the practice of optimizing websites to improve their ranking on search engines.
- It involves improving content quality, keywords, page structure, and website performance.
- SEO also includes technical aspects like mobile responsiveness and page loading speed.
- Good SEO increases organic traffic and helps users find relevant information easily.

### 5. What is SDLC Life Cycle?

- SDLC (Software Development Life Cycle) is a step-by-step process used to develop high-quality software.
- It begins with requirement analysis and planning to understand user needs.
- The lifecycle includes design, development, testing, deployment, and maintenance stages.
- SDLC helps ensure software is delivered on time, within budget, and meets quality standards.

## LAB EXERCISE

### Phases of SDLC Life Cycle

1. **Requirement Analysis** – Collecting and understanding user needs and system requirements.
2. **System Design** – Creating system architecture, design specifications, and workflows.
3. **Development (Coding)** – Writing and building the actual software based on the design.
4. **Testing** – Checking the software for bugs, errors, and performance issues.
5. **Deployment** – Releasing the software to users or production environment.
6. **Maintenance** – Fixing issues, updating features, and improving performance after release.
